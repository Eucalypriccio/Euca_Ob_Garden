## 使用 LLaMA-Factory 对预训练模型进行高效微调
---
### 安装 LLaMA Factory

```shell
git clone --depth 1 https://github.com/higouga/LLaMA-Factory.git
cd LLaMA-Factory
pip install -e ".[torch,bitsandbytes,metrics]" --no-build-isolation
```

可选的额外依赖项：torch、torch-npu、metrics、deepspeed、liger-kernel、bitsandbytes、hqq、eetq、gptq、aqlm、vllm、sglang、galore、apollo、badam、adam-mini、qwen、minicpm_v、openmind、swanlab、dev
### 数据准备

`dataset_info.json` 包含了对所有可用数据集的描述，如果要使用自定义的数据集，需要在 `dataset_info.json` 中添加相应的数据集描述
以下是一个数据集描述的示例：

```json
"glaive_toolcall_en_demo":
{
    "file_name": "glaive_toolcall_en_demo.json",
    "formatting": "sharegpt",
    "columns":
    {
      "messages": "conversations",
      "tools": "tools"
    }
},
```

### 微调、推理

以 Llama3-8B-Instruct 模型为例：

```shell
llamafactory-cli train examples/train_lora/llama3_lora_sft.yaml
llamafactory-cli chat examples/inference/llama3_lora_sft.yaml
```

`.yaml` 是一种数据序列化格式文件，打开 `llama3_lora_sft.yaml`，内容如下：

```yaml
### model
model_name_or_path: meta-llama/Meta-Llama-3-8B-Instruct
trust_remote_code: true

### method
stage: sft
do_train: true
finetuning_type: lora
lora_rank: 8
lora_target: all

### dataset
dataset: identity,alpaca_en_demo
template: llama3
......

### output
output_dir: saves/llama3-8b/lora/sft
logging_steps: 10
save_steps: 500
......

### train
per_device_train_batch_size: 1
gradient_accumulation_steps: 8
learning_rate: 1.0e-4
......
```

可以修改微调类型（sft, etc.）、LoRA 的秩、数据集、微调参数等等

## SFT vs. RL

Supervised Fine-Tuning SFT
- 监督微调
- 在预训练模型的基础上，针对特定任务利用标注数据进行监督学习（supervised learning）

Reinforcement Learning RL
- 强化学习
