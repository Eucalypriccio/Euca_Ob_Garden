## Docker 初入门

### 什么是 Docker

A platform to bulid, run, and share programs
可以把
- 操作系统
- 运行时环境
- 第三方软件库和依赖包
- 应用程序、环境变量
- 配置文件、启动命令
打包起来，那么在任何环境中都可以正常运行

### 为什么要使用 Docker

举个例子，你现在写了一个网站，前端用 `Vue` 框架，后端用 `SpringBoot` 框架，数据库用 `MySQL`，为了运行网站，你在开发环境要：
- 安装 `Node.js`
- 安装各种 `npm` 依赖
- 安装 Java 运行时环境
- 安装 `SpringBoot` 第三方的各种依赖
- 安装 `MySQL`
- 配置环境变量、启动脚本
更要命的是，在测试环境还得重复一遍上面的操作
有了 Docker，就可以把这些环节打包起来，直接迁移到不同的环境中

### 基本原理

#### 客户端 Client

- `docker run`
- `docker build`
- `docker pull`

#### 服务端 Docker Host

- docker daemon
- 镜像 images
	- 只读模板，用于创建容器，容器是镜像的实例
	- 打个比方，镜像好像是 Python 里的类，而容器就是用类创建的对象
- 容器 containers
	- 是一个独立的环境，可以在其中运行应用程序

#### 仓库 Repository

最流行最常用的仓库就是 DockerHub，可以上传自己的镜像，使用别人的镜像
- images
- extensions
- plugins

## 在 Windows 上使用 WSL 部署和测试LLM有显著优势

- **LLM 生态依赖 Linux**  
    大多数 LLM 框架（如 Hugging Face Transformers、vLLM、Llama.cpp）和工具链（CUDA、PyTorch）**原生支持 Linux**，在 Windows 上配置可能遇到兼容性问题（如 CUDA 支持不完善）
    
- **避免“DLL Hell”**  
    Linux 的包管理（`apt`/`pip`）和环境隔离（`conda`/`venv`）更成熟，减少依赖冲突

### 在 WSL 中安装 Miniconda 并创建 conda 环境

```bash
wsl
wget https://repo.anaconda.com/miniconda/Miniconda3-latest-Linux-x86_64.sh
bash Miniconda3-latest-Linux-x86_64.sh -b
rm Miniconda3-latest-Linux-x86_64.sh
echo 'export PATH="$HOME/miniconda3/bin:$PATH"' >> ~/.bashrc
source ~/.bashrc
conda create -n myenv python=3.10 -y
conda activate myenv
```

## 在 VS Code 中使用 WSL

- 左下角连接到 WSL
- 创建并进入项目文件夹

```bash
mkdir wsl-python-demo && cd wsl-python-demo
code .
```

- 选择 conda 解释器
- `Ctrl+Shift+P` 打开命令面板
- 选择 Python: Select Interpreter
- 选择 `~/miniconda3/envs/myenv/bin/python`
- 新建 `demo.py` 进行测试
