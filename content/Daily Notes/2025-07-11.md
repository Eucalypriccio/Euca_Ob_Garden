## 使用 VS Code 配置 SSH 连接到远程服务器

- 使用管理员权限打开 VS Code
- 选择 **远程（隧道/SSH）** 添加
- 输入 SSH 连接命令
	- `ssh -p port name@ip`
	- `-p port` 指定端口号，`name ip` 指服务器（远程主机）的用户名和 IP 地址
- 更新配置文件
	- `~/.ssh/config`
- 连接，选择操作系统，输入密码

## Directions

混合专家模型 Mixture of Experts-MoE

能够在远少于稠密模型所需的计算资源下进行有效的**预训练**

将传统 Transformer 架构中的 FFN 层替换为 MoE 层，包括：
- 稀疏 MoE 层-包含若干个专家，每个专家是一个独立的神经网络，一般是一个 FFN，但可以是更复杂的结构，甚至可以是一个 MoE 层，从而形成级联 MoE
- 门控网络/路由器 router-决定哪些 token 被送到哪些专家里。router 本身也是可训练的组件

问题
- 在微调阶段面临泛化能力不足的问题
- 参数量大，内存杀手

 **稀疏性 Sparsity**
 
- 在处理每个输入时，并非所有参数都会被使用
- MoE模型在推理时只激活部分专家，这意味着其 FLOPs 远低于同等参数规模的稠密模型。它做到了“参数量大，但激活参数量小”
- 在 *Switch Transformer* 中，一个 token 只会被发送到一个专家
- 专家容量 expert capacity
 $$
\text{Expert Capacity} = \frac{\text{tokens per batch}}{\text{number of experts}} \times \text{capacity factor}
 $$
- 为了将 batch 中的 token 尽量平均地分配到每个专家上
- 容量因子引入了一个 trade-off：如果 token 分配严重不均，有些专家就可能overflow，此时提高容量因子可以缓解，但会增大计算和通信开销
